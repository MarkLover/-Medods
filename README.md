# Task Service (Extended)

Сервис для управления задачами с поддержкой **периодических (повторяющихся) задач**.

## Особенности реализации (New Features)

В проект добавлена система планирования задач, которая позволяет врачам и персоналу автоматизировать рутинные действия.

### 1. Периодичность (Recurrence)
Теперь при создании задачи можно указать правила её повторения:
- **Ежедневно:** каждые `n` дней.
- **Ежемесячно:** в конкретные числа месяца (например, 1-е и 15-е).
- **Конкретные даты:** создание задач на точно указанный список дат.
- **Четные/Нечетные дни:** автоматическая генерация по календарным дням.

### 2. Срок выполнения (ScheduledAt)
Добавлена поддержка поля `scheduled_at`. Оно позволяет задать время начала задачи. Для обычных задач это "дедлайн", для периодических — точка отсчета для генерации.

## Технические решения (Assumptions)

В ходе разработки были приняты следующие архитектурные решения:

1. **Синхронная предгенерация (Pre-generation):** 
   При создании периодической задачи система сразу рассчитывает все будущие экземпляры и сохраняет их в БД. 
   - *Почему:* Это гарантирует, что задачи будут видны в списках сразу, не требуя фоновых планировщиков (cron-воркеров).
   - *Лимит:* Если не указана дата окончания (`end_date`), задачи генерируются на **1 год вперед**.

2. **Независимое управление:** 
   Каждый экземпляр повторяющейся задачи в базе данных является независимым. Это позволяет врачу перенести или удалить одну конкретную операцию в цепочке, не затрагивая остальные.

3. **Оптимизация производительности (Bulk Insert):**
   При генерации задач на год вперед используется механизм батчевой вставки (`pgx.Batch`). Это позволяет создать сотни записей в БД за один запрос, обеспечивая моментальный HTTP-отклик.

## Быстрый запуск

### Требования
- Go `1.23+`
- Docker и Docker Compose

### Запуск проекта
```bash
# Если вы запускали проект ранее, очистите старые данные для применения миграций:
docker compose down -v 

# Запуск:
docker compose up --build
```
Сервис будет доступен по адресу `http://localhost:8080`.

---

## API & Документация

### Swagger / OpenAPI
- **Swagger UI:** `http://localhost:8080/swagger/`
- **OpenAPI JSON:** `http://localhost:8080/swagger/openapi.json` (обновлен с учетом новых полей).

### Основные эндпоинты
- `POST /api/v1/tasks` — создание (поддерживает объект `recurrence`).
- `GET /api/v1/tasks` — список всех задач (включая сгенерированные).
- `PUT /api/v1/tasks/{id}` — редактирование конкретного экземпляра.
- `DELETE /api/v1/tasks/{id}` — удаление.

## Тестирование
Для проверки логики генерации дат выполнены Unit-тесты:
```bash
go test ./internal/usecase/task/...
```
Тесты покрывают все типы периодичности и краевые случаи (конец месяца, високосные годы и т.д.).
